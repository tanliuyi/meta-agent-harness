# 设计记录

## 整体视觉语言

范围：桌面端 renderer UI。

- 设计方向是简洁、像素、赛博、机能；首先保证信息密度、层级和操作效率，再考虑风格装饰。
- 使用硬朗的小圆角体系。优先使用设计 token，避免在组件内重新引入大圆角、胶囊形状或圆形主操作按钮。
- 边框是主要层级手段：普通控件使用轻边框，重要容器或主操作可使用强边框；避免同时叠加粗边框、重内阴影和重投影。
- 阴影用于区分浮层与页面，不用于制造明显的凹陷效果。Input 和 Textarea 应与 Select 保持接近的视觉深度，使用轻微内阴影即可。
- 动效应短促直接。按钮按下使用轻微纵向位移；状态动画可使用离散帧，但必须支持 `prefers-reduced-motion`。
- 中文正文和标题默认使用 UI 字体。等宽字体用于代码、协议值、状态帧或确有终端语义的短标签，不应为了风格强行应用于普通中文。

## 像素元素

- 像素元素必须表达层级、状态或过程，不能只是填补空白的装饰。
- Dialog 主标题可使用双格像素标记，内部一级分区标题可使用单格标记，以形成明确的层级关系。
- 运行状态可以使用分帧像素动画，例如 Chat activity 的三个像素点。
- Toast、普通通知卡片和列表项角落不添加无语义的孤立像素点。
- 如果像素概念不能自然融入组件，应保持简洁，不额外添加方块、短线或扫描效果。

## 按钮与操作

- 主按钮使用强化边框、小圆角和较高字重；按下反馈为 `translateY(1px)`，避免缩放导致布局感知不稳定。
- 次级按钮通过表面色和强边框区分；ghost 按钮默认透明，只在 hover/active 时显示轻量背景。
- 熟悉的编辑、删除、关闭、刷新等操作优先使用图标按钮，并提供可访问名称与 Tooltip。
- 同一内容项的操作应放在独立操作行或稳定的操作区，不与正文争抢横向空间。
- 危险操作使用 danger 颜色，但默认状态应克制，hover 后再增强反馈。

## 表单控件

- Input、Textarea 和默认 Select 应保持一致的边框层级、控件高度和聚焦反馈。
- Input 和 Textarea 使用 1px 边框与极轻内阴影，避免明显的内凹或厚重描边。
- Focus 状态使用 primary 边框和外部 focus ring；不要通过加深内阴影表达焦点。
- 表单标签使用较小字号和稳定字重；帮助文本弱化，不与输入值竞争视觉权重。
- JSON、代码和协议配置可以使用等宽字体；普通名称、说明与中文内容继续使用 UI 字体。

## Dialog

- 长表单 Dialog 采用固定头部、可滚动主体和固定底部操作栏，避免整个窗口随内容滚动。
- Dialog 尺寸应与任务匹配。Provider 编辑器属于中大型表单，Memory 编辑器属于中型编辑窗口，不默认占满桌面。
- 头部负责标题和简短说明，主体负责内容，底部只放取消与主提交操作。
- 高级或低频配置应折叠，优先保证常用字段在首屏可见。
- 避免卡片套卡片。分区主要通过间距、分隔线、背景层级和标题组织。
- Dialog 使用小圆角和清晰边框；投影只用于建立浮层层级，不使用大面积霓虹或复杂扫描装饰。

## Toast 与会话通知

- Toast 是紧凑的全局反馈条，由状态图标、标题、可选描述和关闭按钮组成。
- 状态颜色由图标承担；不额外添加无语义色条或角落像素点。
- Toast 使用轻边框、轻投影和自然 UI 字体，长描述允许换行，标题保持易扫读。
- 会话通知位于 Composer 上方，宽度随内容自适应；短消息不应撑成大卡片。
- 通知动画只使用轻微位移和透明度变化，避免缩放和夸张弹跳。

## Composer

- Composer 是高频工作区，应保持紧凑、稳定且弱装饰；输入区、附件区和操作栏通过间距与细分隔线组织。
- Focus 状态强化边框与轻量 halo，不改变组件尺寸。
- 附件、引用和模式标签使用小圆角硬边控件，不使用胶囊样式。
- 模型、Thinking 和交付方式属于内联控制，使用 borderless Select，不能抢占输入内容的视觉焦点。
- 发送和停止是主操作，可以使用更强的颜色、边框和轻微硬阴影，但尺寸必须稳定。
- 命令面板和补全列表是浮层工具，选中项通过浅背景和边框表达，不使用重阴影或大圆角列表项。

## 高密度列表

- Memory、Session Tree 等工具列表优先使用连续分隔布局，不将每条记录包装成独立浮动卡片。
- 条目 hover 可提升表面色；active 状态使用 primary soft 背景和明确的边缘状态。
- 正文与操作分层：内容保持完整宽度，操作放在单独一行或固定区域。
- 桌面端可在 hover/focus 时显示次要操作；触屏环境必须保持操作可见。
- 工具栏使用紧凑排列和边界分隔，搜索、筛选、计数和主操作应有稳定位置。
